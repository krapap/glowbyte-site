## Git / деплой

- Репозиторий инициализирован локально, подключён `origin = github.com/krapap/glowbyte-site`.
- Была проблема: при ручной загрузке через сайт папка `assets/` задвоилась во вложенную папку → стили не грузились (404). Исправлено принудительным `git push -f` чистой локальной версии.
- Дальше обновляем только через git:
  ```bash
  cd ~/glowbyte-site
  git add -A && git commit -m "что изменил" && git push
  ```

## Версионирование CSS/JS (сброс кеша)

- Проблема: браузер кешировал `assets/site.css`, и после обновления стилей страницы показывали старую версию (например, на странице статьи не применялись стили — «слипшаяся» мета-строка, текст на всю ширину).
- Решение: ко всем ссылкам на общие ассеты приписан query-параметр версии — `assets/site.css?v=2`, `assets/site.js?v=2` (во всех HTML, кроме `index.html` — там стили инлайн).
- **Правило на будущее:** после каждой правки `site.css`/`site.js` поднимать версию — `?v=3`, `?v=4` и т.д. Тогда все посетители сразу получат свежий файл без жёсткой перезагрузки.
  - Быстро поднять версию во всех файлах (пример: 2 → 3):
    ```bash
    cd ~/glowbyte-site
    sed -i '' -E 's@(site\.(css\|js))\?v=2@\1?v=3@g' *.html
    ```
